vidyeet-gui
=============

Windowsデスクトップ向けの軽量GUIクライアントで、Muxの動画アップロードと管理をローカルで簡単に行えるように設計されています。

主なポイント
- 目的: Mux APIへ動画をアップロード・一覧表示・再生・削除する操作をGUI化する
- 対象: Windows（配布物もWindows向け）
- アーキテクチャ: Vue 3 (Renderer) + Electron (Main + Preload) + CLIラッパー (`bin/vidyeet-cli.exe`)

特徴（ユーザー向け）
- 認証管理: 起動時に認証状態を確認し、未認証ならログイン画面を表示
- ライブラリ表示: サムネイル/アニメGIF/HLSによるプレビューと基本メタデータ表示
- アップロード: ファイル選択ダイアログまたはドラッグ&ドロップでアップロードを開始、進捗を分かりやすく表示
- 削除操作: 確認ダイアログを介して安全に削除
- シェア機能: 再生用URLやMP4リンクをクリップボードへコピー

素早い開始（開発者向け）
- 必要な環境: Node.js + npm/yarn（Windowsでの実行を想定）
- 開発サーバー: `npm run dev`（`vite`）
- ビルド: `npm run build:dir`

仕組み
- UI (Vue) は直接OS操作やCLIの実行を行わず、Preload経由でMainプロセスへ安全にIPCリクエストを送ります。
- Mainはリソースに同梱された `bin/vidyeet-cli.exe` を子プロセスで起動し、JSON出力をパースして結果をRendererへ返します。
- 認証やファイル管理の継続的な保持はCLIの責務で、GUIは必要最小限の表示と操作を担います。

セキュリティとプライバシー
- 認証情報の保存・管理はCLI側が行います。GUIは成功/失敗などの状態のみを受け取り表示します。
- Preload経由で必要最小限の機能だけをcontextBridgeで公開しており、Rendererの権限は限定されています。
