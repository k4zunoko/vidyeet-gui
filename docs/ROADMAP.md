# Roadmap

## フェーズ0: MVP（現実装に含まれる）

- 起動時 `status` による認証チェック
- 未認証時のログイン導線
- `list` による一覧表示
- サムネイル + ホバーGIF + 選択でHLS再生

フェーズ0に含めた拡張（現実装）:

- ログアウト
- 右クリックメニュー
  - MP4リンクコピー
  - 削除（確認ダイアログあり）
- アップロード（ファイル選択 + 進捗表示 + 完了後リロード）
  - **ノンモーダルダイアログ**: 画面右下固定、他の操作をブロックしない（Nielsen Norman Group準拠）
  - **最小化機能**: アップロード中でもコンパクトバーに縮小可能
  - ドラッグ＆ドロップ対応（複数ファイル対応）
  - **✓ キューシステム**: 複数ファイルの逐次アップロード
    - 複数ファイル選択時に自動的にキューに追加
    - 1ファイルずつ順次処理（エラー発生時も次へ進む）
    - 成功時の個別リロード（完了したファイルをすぐに一覧に追加）
    - 待機中リストの表示とキャンセル機能
    - 実装: `src/composables/useUploadQueue.ts`

## フェーズ1: 基本操作の拡張

- 動画詳細の表示（`show <asset_id>`）
- "URLコピー"の拡張
  - HLS URL
  - サムネイルURL
  - MP4 URL（現状は対応済み）

## フェーズ2: アップロード拡張

- ~~複数ファイルの逐次アップロード（キューシステム）~~ **✓ Phase 1実装済み**
  - 基本キュー機能（追加・逐次実行・キャンセル）
  - 待機中リスト表示
  - 個別リロード
- 失敗時の再試行機能（エラーアイテムの再試行ボタン）

- ~~進捗（percent）の表示強化~~ **✓ 実装済み（進捗補間）**

  - 指数移動平均（EMA）とease-outカーブによる滑らかな補間
  - chunk境界での自然な減速・停止
  - 推定であることのUI明示
  - 実装: `src/composables/useProgressInterpolation.ts`
  - 詳細: `docs/UX_PSYCHOLOGY.md` - 進捗表示の補間セクション

## フェーズ3: 品質/運用

- ログ保存（ユーザー報告を容易に）
- 設定画面（プロキシ・タイムアウト等）※必要になったら

## 優先順位の判断基準

- CLI契約が安定しているか（非対話ログイン/機械可読進捗など）
- UI上のエラーが復帰可能か（ユーザーが自力で解決できる導線があるか）
- 追加機能が既存の責務境界（Main/Preload/Renderer）を壊さないか
